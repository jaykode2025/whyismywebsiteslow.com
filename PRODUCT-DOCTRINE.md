# Product Doctrine — Why Is My Website Slow

## Vision

> **A business-first website diagnostics platform that attracts traffic through high-intent SEO pages, captures proprietary performance data through scans, turns that data into better benchmarks and recommendations, and monetizes through reports, monitoring, and fixes.**

This is **not** a PageSpeed wrapper. This is a **business intelligence layer** for website owners.

---

## The Three-Layer Business

### Layer 1: Traffic Engine
- Programmatic SEO pages (industry, platform, fix, comparison)
- High-intent search queries ("why is my Shopify site slow")
- Shareable report pages with unique proof artifacts
- **Goal**: Bring visitors who need answers, not just tools

### Layer 2: Data Engine
- Every scan enriches proprietary dataset
- Normalized issue taxonomy (image bloat, JS overload, third-party drag, etc.)
- Industry/platform baselines (what's normal for lawyers? for Shopify?)
- Fix outcome history (what improvements follow which fixes?)
- **Goal**: Build assets competitors cannot clone

### Layer 3: Monetization Engine
- Free scan → acquisition & trust
- Paid deep report → low-friction first purchase ($9–$49)
- Monitoring subscription → recurring revenue (alerts, trends)
- Done-for-you fixes → highest cash offer (agencies, implementations)
- Benchmark intelligence → premium data product
- **Goal**: Multiple revenue paths from same traffic

---

## The Moat (7 Assets)

1. **Scan dataset** — More sites = better benchmarks
2. **Issue taxonomy** — Organized noise into useful categories
3. **Industry baselines** — What's normal for dentists vs. lawyers vs. Shopify
4. **Fix outcome history** — "Sites with X improved Y% after these fixes"
5. **Page/report inventory** — Discovery graph across problems
6. **Internal linking graph** — Route users: problem → diagnosis → solution
7. **Brand trust** — Plain-English business translation, not engineer-speak

## Page Archetypes (Each Must Feel Different)

| Type | Purpose | Proof Block | CTA |
|------|---------|-------------|-----|
| **Diagnosis** | Explain what's wrong | Scorecard + top issues | Unlock full report |
| **Benchmark** | Compare to peers | Industry comparison chart | Start monitoring |
| **Comparison** | Tool vs. tool | Side-by-side table | Try ours free |
| **Checklist** | Actionable steps | Progress tracker | Download PDF |
| **Case Study** | Show outcomes | Before/after metrics | Get same results |

**No samey pages.** Every page needs:
- Structural variation (section order)
- Visual variation (chart-first vs. table-first)
- Proof variation (unique artifact)
- CTA variation (scan vs. report vs. monitoring vs. service)

---

## Report Page (Most Important Page)

### Above the fold
- URL analyzed
- Overall diagnosis (plain English)
- Strong visual proof block
- Primary CTA

### Below the fold
- Top 3 issues (prioritized)
- Business impact (revenue, trust, SEO)
- Benchmark comparison (vs. industry/platform)
- Fix paths (what to do next)
- Trust/reliability explanation
- Secondary CTA

**Order matters**: Users care about what's wrong → how bad → what it costs → what to do.

## Technical Principles

### Astro + Svelte
- Astro does heavy lifting for HTML/SEO
- Svelte hydrates **only** where it adds value:
  - Charts
  - Mini calculators
  - Interactive filters
  - Scan progress UI
  - Report toggles

### Data Structure
- Clean issue taxonomy
- Benchmark logic by industry/platform
- Structured page metadata (topic, intent, CTA type)
- Report storage supporting comparison & history

### Enhanced Scanner
- CrUX (real-user metrics) + PSI (lab) + network checks
- 60% weight on real-user data, 40% on lab
- Confidence tracking (sample size)
- A-F grade + 0-100 score

### Dynamic OG Images
- Every important page/report gets unique social preview
- Helps trust, click-through, sharing

### Related Links Engine
- Route users naturally: problem → industry → scan → report → offer
- Helps SEO + funnel flow
